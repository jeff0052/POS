## 1. 产品定位

FounderPOS 不是一个传统 POS 收银系统，而是一个 **AI 驱动的餐饮经营操作系统**。

核心理念：**让一个老板 + AI 合伙人，就能管好一家餐厅。**

```
传统 POS：老板 + 10 个员工 + 手工管理 = 勉强运转
FounderPOS：老板 + AI 运营总监 + 精简团队 = 高效运转
```

---

## 2. 系统架构

```
Layer 4 — Agent Identity Layer
           每家餐厅 = 1 Agent + 1 Wallet
           对外：预定/包场/合作/外卖平台对接

Layer 3 — AI Operator Layer
           5 个 AI 顾问：菜单/营销/会员/经营/出品
           模式：Sense → Think → Propose → Approve → Act

Layer 2 — MCP Tool Layer
           每个业务域暴露为 MCP Tools
           人类和 AI 走同一套接口，统一审计

Layer 1 — Transaction Foundation
           订单/结账/退款/库存/会员/促销/报表
           传统业务底座，稳定可审计
```

### 3.11 老板 (OWNER)

### 4.2 三类菜单入口

**现状：⚠️ 30%（只有单点）**

#### 4.2.1 单点菜单（已完成）

#### 4.2.2 自助餐菜单（待开发）

- 展示自助档位（22 元/38 元/58 元）

- 套餐内商品免费或有差价
- 套餐外商品按原价额外收费

#### 4.2.3 外卖菜单（待开发）

- `delivery_platform_configs` — 平台 API 配置
- `delivery_orders` — 外卖订单扩展

### 4.5 多门店多价格

**现状：⚠️ 30%**

- 同一个 SKU 在不同门店不同价格
- 同一个 SKU 在不同渠道不同价格
- 同一个 SKU 在不同时段不同价格
- 同一个 SKU 对不同会员等级不同价格

**价格解析优先级：**
```
门店+场景 > 品牌+场景 > 门店基础价 > SKU 默认价
```

### 4.7 CRM 会员体系

**现状：⚠️ 40%（基础注册/积分/充值有，完整体系缺）**

#### 等级规则（可配置）
- 升级条件：累计消费/周期消费/次数
- 降级规则：可选开启
- 权益：积分倍率、折扣、生日奖励、免运费

#### 优惠券
- 满减/折扣/赠菜/免运费 四种类型
- 固定日期/领后N天 两种有效期
- 按门店/品类/SKU/用餐模式限定适用范围
- 系统发/活动领/充值送/生日送/推荐奖/积分兑 六种来源

#### 储值
- 充值活动：充 X 送 Y（余额 + 积分 + 券）
- 储值支付：直接当钱花
- 储值流水：完整审计

#### 积分（独立于储值）
- 10+ 赚取规则：消费赚、多倍日、品类加倍、生日、签到、评价
- 分批过期：每次赚的积分独立过期时间，FIFO 先到期先扣
- 抵扣规则：汇率（100积分=1元）、最多抵 50%、排除品类
- 兑换：积分换菜品/换券

#### 推荐裂变
- 每个会员有推荐码/链接/二维码
- 被推荐人首单后触发奖励
- 推荐人和被推荐人都得奖励

#### 会员标签
- 自动标签：高价值/流失风险/常客/新客/即将生日
- 手动标签：自定义
- 按标签精准营销

#### 消费画像
- 定期聚合：偏好时段/品类/SKU/支付方式/门店
- 用于推荐和营销

#### 营销触达
- WhatsApp/SMS/Email 批量推送
- 活动类型：促销/召回/生日/新品
- 按标签/等级/自定义条件筛选人群
- 转化追踪

## 8. 数据模型

**总计 119 张表，20 个模块。**

详见：
- `docs/75-complete-database-schema.md` — 完整 DDL
- `docs/76-database-schema-readme.md` — Schema 导航

### 核心设计原则

1. **SKU 为核心** — 三层模型（顾客/厨房/库存）
2. **价格可覆盖** — `sku_price_overrides` 支持无限场景
3. **积分分批过期** — `points_batches` FIFO
4. **库存分批追踪** — `inventory_batches` FIFO
5. **权限可自定义** — 商户自建角色 + 权限积木
6. **渠道可追踪** — 每笔订单归因 + 分润计算
7. **所有金额用分** — BIGINT cents，不用 DECIMAL
