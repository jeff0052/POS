# Milestone: 2026-03-27

## One-Line Summary

From prototype to production-grade architecture: real auth, payment microservice, AI layer, 6-service deployment on AWS — in one day.

---

## What Was Delivered

### 1. Full-Stack Architecture Audit & Fix
- Audited entire codebase (backend, Android, 4 frontends, infra)
- Found 8 CRITICAL, 12 HIGH, 17 MEDIUM, 5 LOW issues
- Fixed all CRITICALs across 6 rounds of Codex audit/re-review

### 2. Real Authentication & Authorization
- JWT auth with BCrypt password hashing
- Spring Security filter chain with role-based access
- PC admin: token storage + Bearer injection + 401 auto-logout
- Platform admin: login page + route guard
- Bootstrap endpoint (secret-gated, concurrent-safe) for first admin setup
- No more mock tokens or hardcoded users

### 3. Payment Microservice (pos-payment-service)
- Independent Spring Boot service on port 8081
- Own database (pos_payment_db) with Flyway migrations
- PaymentProviderAdapter interface for pluggable providers
- CashAdapter working end-to-end (create intent → succeed → callback POS → settle)
- HMAC-signed callback to POS backend with 3x retry + backoff
- SETTLEMENT_FAILED status when callback exhausted (no silent data loss)
- API key authentication (X-Service-Key)
- Fail-fast: refuses to start without secrets
- External team handoff document (docs/55)

### 4. AI Layer (3 modules)
- **MCP Tool Server**: 14 tools across 6 domains (catalog, order, member, promotion, settlement, report)
- **AI Operator**: 5 advisor roles (Menu, Marketing, CRM, Operations, Kitchen), recommendation lifecycle with approval flow
- **Agent + Wallet**: restaurant identity, wallet ledger, risk-based interaction routing

### 5. Business Modules
- Staff / Role / Permission with PIN verification
- Cashier shift management (open/close/reconciliation)
- GTO tax export (Singapore GST 9/109 extraction)
- Member lifecycle (registration, recharge, points earning, tier auto-upgrade)
- Promotion engine enhancement (percent discount, usage limits)
- Refund flow with pessimistic locking

### 6. Security Hardening
- Server-side price lookup (POS no longer trusts client prices)
- Webhook signature mandatory (VibeCash)
- Settlement amount validation (cash ≥ payable, digital = exact)
- MCP endpoint locked behind ADMIN role
- Payment API blocked from public nginx
- Old unvalidated collect endpoint removed
- SecurityException → 403 (not 409)

### 7. Platform Admin (Total Backend)
- 7 pages: Dashboard, Merchants, Stores, Devices, Configurations, Users, Monitoring
- Real API data (store count from DB)
- Login + auth guard

### 8. Infrastructure
- AWS deployment: 8 containers (mysql, backend, payment, nginx, pc-admin, pos-frontend, qr-frontend, platform-admin)
- docker-compose.prod.yml: complete, all services defined
- docker-compose.yml: local dev with all secrets
- CI: 6 jobs (backend, payment service, 4 frontends)
- Nginx: CSP headers, per-frontend routing, payment API blocked
- Maven wrapper for both Java services
- .env.example with all required secrets
