## 1. 目标

订单提交后按 SKU 工作站路由拆票，KDS 状态流转，离线自动回退打印。

## 2. 设计决策

| # | 决策 | 选择 | 原因 |
|---|------|------|------|
| K1 | 拆票触发方式 | 同步调用（同事务） | 简单可靠，失败整笔回滚 |
| K2 | 心跳监控方式 | 被动检查（拆票时检查） | 无需定时任务，简单可靠 |
| K3 | 退单库存回补 | 只做票状态 CANCELLED，留 TODO | 库存模块在 Session 4.2 |
| K4 | enforceStoreAccess | 提取 StoreAccessEnforcer 共享组件 | 解决 Phase 3 遗留 #4 |
| K5 | SecurityConfig | 不动，权限在 service 层检查 | 与 Phase 3 保持一致 |

## 4. 状态机

```
kitchen_tickets.ticket_status:

  SUBMITTED ──→ PREPARING ──→ READY ──→ SERVED
       │              │
       └──────────────┴──→ CANCELLED
```

**转换规则：**

| 当前状态 | 允许的目标状态 | 所需权限 |
|---------|--------------|---------|
| SUBMITTED | PREPARING | KDS_OPERATE |
| PREPARING | READY | KDS_OPERATE |
| READY | SERVED | KDS_OPERATE |
| SUBMITTED | CANCELLED | TICKET_CANCEL |
| PREPARING | CANCELLED | TICKET_CANCEL |

## 5. 核心组件

### 5.1 Entity 层

**KitchenStationEntity** (`kitchen` 包)
- 映射 `kitchen_stations` 全部字段（含 V095 fallback 字段）
- `@Entity(name = "V2KitchenStationEntity")` 避免与可能的旧 Entity 冲突
- `isOnline()` → `"ONLINE".equals(kdsHealthStatus)`
- `isHeartbeatExpired(int timeoutSeconds)` → `last_heartbeat_at + timeout < now()`
- `markOffline()` / `markOnline()`

**KitchenTicketEntity** (`kitchen` 包)
- 映射 `kitchen_tickets` 全部字段
- `@OneToMany` 关联 `KitchenTicketItemEntity`
- 状态转换方法内置校验：`transitionTo(newStatus)` 抛 `IllegalStateException`

**KitchenTicketItemEntity** (`kitchen` 包)
- 映射 `kitchen_ticket_items`
- `option_snapshot_json` 用 `@Column(columnDefinition = "JSON")`

### 5.2 Repository 层

| Repository | 关键查询 |
|-----------|---------|
| JpaKitchenStationRepository | `findByStoreIdAndStationStatusOrderBySortOrderAsc(storeId, status)` |
| | `findByStoreIdAndSortOrder(storeId, 0)` — default station |
| JpaKitchenTicketRepository | `findByStoreIdAndStationIdAndTicketStatusIn(storeId, stationId, statuses)` |
| | `findBySubmittedOrderId(orderId)` |
| JpaKitchenTicketItemRepository | （通过 ticket cascade，一般不直接用）|

### 5.3 Service 层

**TicketRoutingService** — 拆票引擎
```
routeOrder(SubmittedOrderEntity order) → List<KitchenTicketEntity>

1. 查 order.items 的所有 skuId → 批量查 SkuEntity 获取 station_id
2. 按 station_id 分组（null station_id → default station）
3. 每组生成 KitchenTicket:
   - ticket_no = "KT-{storeId}-{yyyyMMddHHmmss}-{seq}"
   - table_id, table_code 从 order 取
   - round_number 从 order 的同 session 已有 ticket 数量 + 1 推算
4. 拆票前被动检查目标 station 心跳:
   - last_heartbeat_at > 90s → markOffline()
   - OFFLINE + fallback_mode=AUTO → 票上标记 fallback_printer_ip
5. 保存所有 tickets（cascade 保存 items）
6. 返回创建的 ticket 列表
```

**TicketStatusService** — 状态流转
```
updateStatus(ticketId, newStatus, storeId) → KitchenTicketDto

1. enforceStoreAccess(storeId)
2. 加载 ticket，校验 ticket.storeId == storeId
3. 权限检查：CANCELLED 需要 TICKET_CANCEL，其他需要 KDS_OPERATE
4. 调用 ticket.transitionTo(newStatus)
5. 填充时间戳：PREPARING→started_at, READY→ready_at, SERVED→served_at
6. 保存
7. CANCELLED 时：// TODO: Session 4.2 库存回补
```

**KdsHeartbeatService** — 心跳
```
heartbeat(stationId, storeId) → StationHeartbeatDto

1. enforceStoreAccess(storeId)
2. enforcePermission(KDS_OPERATE)
3. 加载 station，校验 store
4. 更新 last_heartbeat_at = now()
5. 如果之前 OFFLINE → markOnline()
6. 返回 {kdsHealthStatus, lastHeartbeatAt}
```

**KitchenStationService** — 工作站管理
```
createStation(storeId, request) → StationDto
  - enforceStoreAccess + enforcePermission(KDS_MANAGE)
  - 唯一性校验 (store_id, station_code)

listStations(storeId) → List<StationDto>
  - enforceStoreAccess + enforcePermission(KDS_OPERATE)
  - 返回 ACTIVE 状态的 stations
```

### 5.4 StoreAccessEnforcer（共享组件）

位置：`com.developer.pos.v2.common.application.StoreAccessEnforcer`

```java
@Component
public class StoreAccessEnforcer {
    private final JpaStoreLookupRepository storeLookupRepository;

    public void enforce(Long storeId) {
        AuthenticatedActor actor = AuthContext.current();
        if (actor.merchantId() != null && actor.merchantId() != 0L) {
            StoreEntity store = storeLookupRepository.findById(storeId)
                .orElseThrow(() -> new IllegalArgumentException("Store not found: " + storeId));
            if (!Objects.equals(store.getMerchantId(), actor.merchantId())) {
                throw new SecurityException("Store does not belong to your merchant");
            }
            if (actor.accessibleStoreIds() != null && !actor.accessibleStoreIds().isEmpty()
                    && !actor.hasStoreAccess(storeId)) {
                throw new SecurityException("You do not have access to store: " + storeId);
            }
        }
    }

    public void enforcePermission(String permissionCode) {
        AuthenticatedActor actor = AuthContext.current();
        if (!actor.hasPermission(permissionCode)) {
            throw new SecurityException("Missing permission: " + permissionCode);
        }
    }
}
```

Kitchen 模块全部注入 StoreAccessEnforcer。现有 4 个 service 的 enforceStoreAccess() 本 session 不改（渐进替换，避免引入无关变更）。

### 5.6 Controller

**KitchenV2Controller** — 合并所有 kitchen 端点

```
@RestController
@RequestMapping("/api/v2")
public class KitchenV2Controller implements V2Api {

    POST /stations/{stationId}/heartbeat
      → KdsHeartbeatService.heartbeat(stationId, extractStoreId())

    PUT /kitchen-tickets/{ticketId}/status
      → TicketStatusService.updateStatus(ticketId, request.newStatus, request.storeId)

    GET /stores/{storeId}/kitchen-tickets?stationId=X&status=Y
      → KitchenTicketQueryService.listTickets(storeId, stationId, status)

    POST /stores/{storeId}/kitchen-stations
      → KitchenStationService.createStation(storeId, request)

    GET /stores/{storeId}/kitchen-stations
      → KitchenStationService.listStations(storeId)
}
```
